Demo workflow

This demo is a list of the new modern Web features: "HTML5 & Friends".

  • The first view is a list of different categories, matching w3c.org list, including w3c logos.

  • Click on a category, you'll see the list of the features.

  • Click on a feature, you'll see a popup with a short description of the feature, and, for some features, a demo.

  • Click on the close button or press back, the popup will close.

Favorites

To mark a feature as "favorite": once the popup is displayed, click on the title. Click again to remove it.

This feature will be available in the "Your favorites" for quick access.

The 10 demos.

Device Access → Geolocatoin API

Make sure to test the demo first, not sure coords are available where you are Click on "Share", see the google map.

Device Access → File API

So this one is tricky and really need to be understood:

This demo shows more than just the File API.

  • Click on the placeholder
  • Select a picture (some phones allow you to take a picture)
  • Once clicked, you will maybe have to wait a couple of seconds. This is because Firefox reads the file. So don't select a huge file.
  • The file is rendered in a <canvas> element
  • Click on the picture
  • With Javascript and Canvas, we apply a Sepia filter (a nice Black&White filer)
  • Click again, the picture is automaticaly uploaded to IMGUR.COM. Here, we see 3 interesting things:
    • XMLHttpRequest Level 2 (upload feedback)
    • FormData (super simple way to send data)
    • Cross-Domain XMLHttpRequest (from paulrouget.com to IMGUR.COM)
  • Click again, a new tab is opened. See the image hosted on imgur.com

Tips

Test all the demos before showing them to anybody

Make the demos you want to show as favorite.

You can show a demo in a "fullscreen" mode. To do so, select the demo to show and turn your phone in landscape mode. It can be a bit buggy, if you don't see the demo, just move a bit the page, it will appear.

What's the demo show?

Beside the sub-demos, the demo itself include interesting features:
  • CSS3 everywhere (shiny colors, shadow, favorite mechanism)
  • CSS3 Transition (when you open a menu)
  • localStorage to save the favorites
  • History API to close a demo when you press back
  • HTML5 parser: the categories logo are actually inline SVG
  • MediaQuery for the fullscreen mode
